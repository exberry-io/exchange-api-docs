# Asset

## Create Asset

This API is used to create  the list of assets in the system.

{% hint style="info" %}
\[POST] {Admin REST}/api/assets
{% endhint %}

### Request

<table><thead><tr><th width="186.0869140625">Parameter</th><th width="113.95247395833331">Type</th><th>Description</th></tr></thead><tbody><tr><td>type</td><td>eNum</td><td><p>Asset Type</p><p></p><p>Allowed values:</p><p>- Commodity</p><p>- Cash</p><p>- Other</p></td></tr><tr><td>name</td><td>String</td><td>Asset name</td></tr><tr><td>description</td><td>String</td><td>Asset Description</td></tr><tr><td>inOutUnits</td><td>String</td><td>Units used for Deposits and Withdrawals/ Redemptions</td></tr><tr><td>transactionsUnits</td><td>String</td><td>Units used for Ledger Transactions</td></tr><tr><td>conversionFactor</td><td>Decimal</td><td>Conversion factor from deposit units to transaction units.<br><br>Example: 32.148 means that 1 in/ out unit = 32.148 transaction units.</td></tr><tr><td>precision</td><td>Int</td><td>Transaction Precision<br>Max value: 10</td></tr><tr><td>minDeposit <code>Optional</code></td><td>Decimal</td><td>Minimum Deposit amount in inOutUnits</td></tr><tr><td>minWithdrawal <code>Optional</code></td><td>Decimal</td><td>Minimum Withdrawal amount in inOutUnits</td></tr><tr><td>minTransfer <code>Optional</code></td><td>Decimal</td><td>Minimum Transfer amount in transactionsUnits</td></tr><tr><td>inOutIncrement <code>Optional</code></td><td>Decimal</td><td><p>In out increment in inOutUnits. Deposits and Withdrawal amounts should be multiples of inOutIncrement.</p><p></p><p>For example, if inOutIncrement=5, values such as 5, 10, 15 are allowed and 2, 7, 8 are not allowed.</p></td></tr><tr><td>transferIncrement <code>Optional</code></td><td>Decimal</td><td>Transfer increment in transactionsUnits. Transfer amount of that asset should be multiple of transferIncrement</td></tr></tbody></table>

### **Response**

Same as the request with additional fields below.

<table><thead><tr><th width="186.0869140625">Parameter</th><th width="113.95247395833331">Type</th><th>Description</th></tr></thead><tbody><tr><td>id</td><td>Int</td><td>Asset id generated by the system</td></tr></tbody></table>

### **Error Codes**

<table><thead><tr><th width="128">Code</th><th>Message</th></tr></thead><tbody><tr><td>1</td><td><code>System is unavailable</code></td></tr><tr><td>100</td><td><code>Missing or invalid parameter: [FieldName]</code></td></tr><tr><td>101</td><td><code>[FieldName] must be less than or equal to maxLong/10^ fieldPrecision</code><br><code>Precision of [Fieldname] is greater than [Asset.precision]</code><br><code>[Fieldname] should be an increment of [FieldIncrement]</code></td></tr><tr><td>102</td><td><code>[FieldName] already exists</code></td></tr><tr><td>111</td><td><code>Assets can exist only when LEDGER is enabled</code></td></tr><tr><td>10001</td><td><code>Permission denied</code></td></tr></tbody></table>

### **Samples**

{% tabs %}
{% tab title="Request" %}
```json
curl --location 'https://admin-api-master.rnd.exberry-rnd.io/api/assets' \
--header 'Content-Type: application/json' \
--header 'Authorization: ••••••' \
--data '{
   "name":"GoldXXX",
   "type":"Commodity",
   "description":"Gold 99.99%",
   "inOutUnits":"kg",
   "transactionsUnits":"oz",
   "conversionFactor":"32.148",
   "precision":"5"
}'
```
{% endtab %}

{% tab title="Success Response" %}
```json
{
   "id": "152",
   "name":"GoldXXX",
   "type":"Commodity",
   "description":"Gold 99.99%",
   "inOutUnits":"kg",
   "transactionsUnits":"oz",
   "conversionFactor":"32.148",
   "precision":"5"
}
```
{% endtab %}

{% tab title="Failure Response" %}
```json
{
    "code": "102",
    "message": "name already exists"
}
```
{% endtab %}
{% endtabs %}

## Update Asset

This API is used to update an existing asset in the system.

{% hint style="info" %}
\[PUT] {Admin REST}/api/assets/{Asset.id}
{% endhint %}

### Request

Similar to Create Asset

### **Response**

Similar to Create Asset



### **Error Codes**

Similar to Create Asset with additional errors below.

<table><thead><tr><th width="128">Code</th><th>Message</th></tr></thead><tbody><tr><td>101</td><td><code>[FieldName] not found</code><br><code>Precision can’t be changed: [Account.name].[Asset.name].[FieldName]=[FieldName.value]</code><br><code>Can’t change [FieldName] when active [entity] exist</code></td></tr></tbody></table>

### **Samples**

{% tabs %}
{% tab title="Request" %}
```json
curl --location --request PUT 'https://admin-api-master.rnd.exberry-rnd.io/api/assets/152' \
--header 'Content-Type: application/json' \
--header 'Authorization: ••••••' \
--data '{
   "name":"Gold",
   "type":"Commodity",
   "description":"Gold 99.99%",
   "inOutUnits":"kg",
   "transactionsUnits":"kg",
   "conversionFactor":"1",
   "precision":"6"
}'
```
{% endtab %}

{% tab title="Success Response" %}
```json
{
   "name":"Gold",
   "type":"Commodity",
   "description":"Gold 99.99%",
   "inOutUnits":"kg",
   "transactionsUnits":"kg",
   "conversionFactor":"1",
   "precision":"6"
}
```
{% endtab %}

{% tab title="Failure Response" %}
```json
{
    "code": "101",
    "message": "Asset not found"
}
```
{% endtab %}
{% endtabs %}

## Get Assets

This API is used to retrieve the list of assets in the system.

{% hint style="info" %}
\[GET] {Admin REST}/api/assets
{% endhint %}

### Request

No Request parameters.

### **Response**

An array of `assets`. Each asset object is similar to the response of Create Assets.



### **Error Codes**
